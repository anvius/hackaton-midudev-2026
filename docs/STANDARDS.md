# STANDARDS

## Code Structure

- DDD vertical slicing mandatory.
- `infra/` for non-code infrastructure.
- `bin/` for executable scripts.

## Testing Levels

1. Unit
2. Integration
3. Acceptance
4. E2E

## Commit Convention

Format:

`type(scope): short summary`

Allowed types: `feat`, `fix`, `refactor`, `chore`, `test`.
